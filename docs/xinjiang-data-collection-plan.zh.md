## 目标

这份方案用于规划新疆草原数据采集，目标不是只补几张图，而是为三条路线同时准备数据基础：

1. 新疆域上的 server-side teacher 微调和增强
2. `ViT teacher -> 轻量 student` 的蒸馏
3. `Raspberry Pi 5 + Hailo-8` 的视频流边缘部署

因此，数据必须同时服务于：

- 静态图像监督学习
- 多视角和视频时序建模
- 域迁移与部署分布对齐

## 执行摘要

如果只问“新疆到底要补多少”，建议按三档理解：

### 最低可用方案

- `200-400` 个有真值样方
- 每个样方：
  - `3-5` 张静态图，或
  - `1` 段短视频
- 覆盖 `20-40` 个地点

这档数据够做：

- 域偏移判断
- 第一轮新疆微调
- 粗略验证 teacher 是否能迁移

### 稳定训练方案

- `600-1200` 个有真值样方
- 每个样方：
  - `5-10` 张多视角静态图
  - `1` 段 `5-15` 秒短视频
  - `1` 组 biomass 真值
- 覆盖 `40-80` 个地点

这档数据够做：

- 稳定的新疆版本 teacher 训练
- 较可信的 student 蒸馏
- 初步的视频流多帧融合训练

### 面向部署的理想方案

- 在上面真值样方之外，再补：
  - `几千到几万帧` 无标注真实视频
  - 来自未来真实部署设备和真实移动方式

这档数据主要用于：

- student 蒸馏扩容
- 视频流时序融合
- 边缘部署鲁棒性验证

## 为什么不能只补静态图片

如果未来目标只是服务器上继续训大模型，静态图片当然有价值。  
但如果未来要上 `Pi 5 + Hailo-8`，那只补静态图片不够，原因有三个：

- 边缘部署的输入分布是视频流，不是人工挑出来的单张好图
- 多帧融合要依赖真实时间连续数据
- 蒸馏 student 时，真实视频帧能显著扩大 teacher 行为样本覆盖

所以新疆数据采集要同时补：

- 有真值样方
- 多视角图像
- 真实视频流

## 推荐的数据组成

建议把新疆数据集拆成三类。

### A 类：有真值样方集

这是最重要的数据。

每个样方至少要有：

- 样方编号
- 地点编号
- 采样日期
- 图像或视频
- 生物量真值

推荐一份样方包含：

- `5-10` 张静态图
  - 正视
  - 左斜视
  - 右斜视
  - 远一点
  - 近一点
- `1` 段短视频
  - 建议 `5-15` 秒
  - 缓慢扫过或轻微绕行
- `1` 份 biomass 采样真值

这类数据用于：

- 新疆 teacher 微调
- 监督评估
- 校准 student

### B 类：无标注多视角/视频集

这类数据不一定要有 biomass 真值，但最好来自同域、同设备、同部署条件。

建议规模：

- `几千到几万帧`
- 若干百段短视频

这类数据用于：

- teacher 生成 pseudo targets
- student 蒸馏
- Pi 端时序融合验证
- 部署分布对齐

### C 类：真实部署流数据

这类数据最好来自未来真实安装方式：

- 手持巡检
- 三脚架固定
- 车载低速移动
- 低空杆载缓慢扫描

这类数据即使没有真值，也很值钱，因为它直接暴露：

- 视频压缩噪声
- 曝光变化
- 运动模糊
- 视角变化幅度
- 设备抖动

## 需要覆盖哪些维度

新疆数据不能只看样本数量，还必须覆盖分布变化。

至少要覆盖以下维度：

### 1. 区域分布

尽量覆盖多个地理区域，而不是集中在一小片地块。

建议：

- 至少 `20-40` 个地点用于最低可用方案
- 稳定训练阶段提升到 `40-80` 个地点

### 2. 草地类型

需要覆盖：

- 不同优势种
- 不同草种混生结构
- 不同覆盖度
- 不同高度层次

### 3. 退化程度

至少分层覆盖：

- 低退化
- 中退化
- 高退化

### 4. 季节和物候期

建议至少覆盖：

- 生长初期
- 生长旺盛期
- 枯黄或衰退期

### 5. 光照和天气

至少覆盖：

- 晴天直射光
- 阴天漫射光
- 上午/中午/下午不同太阳高度

### 6. 背景和地表条件

尤其要覆盖：

- 裸土
- 石块
- 阴影
- 枯草比例高
- 稀疏斑块

### 7. 设备差异

如果未来打算部署到某类摄像头，就要尽量用同类设备采样。  
不要全靠高质量相机，最后部署时设备分布完全不同。

## 推荐的样方采样设计

不要“见到就拍”，建议做分层抽样。

一个比较稳的采样单元可以是：

- 地点 `location_id`
- 地点内多个样方 `plot_id`
- 每个样方一组多视角图像和视频

每个样方建议最少采：

- `1` 张正拍
- `2` 张斜拍
- `1` 张远景
- `1` 张近景
- `1` 段短视频
- `1` 组真值

更理想的版本：

- `5-10` 张多视角静态图
- `1-2` 段短视频
- 同一地点不同时间重复观测

## 视频怎么采最有价值

如果未来要做 `Pi 5 + Hailo-8` 的多帧融合，这部分非常关键。

视频建议这样采：

- 每段 `5-15` 秒
- 相机缓慢移动，不要剧烈晃动
- 尽量产生轻微视角变化，而不是完全固定
- 保持未来真实部署时的高度和角度

视频采集动作建议包括：

- 平移扫过
- 轻微弧线绕拍
- 前后靠近和拉远

不建议：

- 剧烈抖动
- 过快扫动
- 长时间纯静止

因为：

- 多帧融合要靠“有新信息的相邻帧”
- 纯静止视频的边际价值不大

## 每个样方至少需要哪些字段

建议把字段分成“必须”和“高价值可选”两层。

### 必须字段

- `sample_id`
- `location_id`
- `plot_id`
- `date`
- `time`
- `lat`
- `lon`
- `device_id`
- `image_path` 或 `video_path`
- `Dry_Green_g`
- `Dry_Dead_g`
- `Dry_Clover_g`
- `GDM_g`
- `Dry_Total_g`

### 高价值可选字段

- `height_avg_cm`
- `ndvi`
- `species`
- `cover_ratio`
- `degradation_level`
- `weather`
- `cloudiness`
- `camera_height_cm`
- `camera_pitch_deg`
- `camera_yaw_deg`
- `collector_id`
- `notes`

这些字段即使当前主线模型不用，也很值得保留，因为以后做分析、校准、错误归因会很有帮助。

## 推荐的数据切分方式

新疆数据切分不能按图片随机切。

应该优先按更高层级切分，避免泄漏：

- 按 `location_id`
- 或 `location_id + date`
- 或至少按 `plot_id`

原因：

- 同一样方的多张图和视频帧高度相似
- 如果随机切分，验证分数会虚高

推荐切分：

- 训练集：约 `70%`
- 验证集：约 `15%`
- 测试集：约 `15%`

切分单位优先级：

1. `location_id`
2. `plot_id`
3. `location_id + season`

## 数据质量检查清单

采集后必须做一轮质量检查。

至少检查：

- 文件是否存在
- 文件名是否唯一
- 样本与标签是否一一对应
- `sample_id` 是否重复冲突
- 视频是否损坏
- 图像分辨率是否异常
- 时间和 GPS 是否缺失
- 真值范围是否异常
- 五个 biomass target 是否满足基本数值逻辑

还建议抽查：

- 同一样方的多张图是否真的是同一样方
- 视频是否真的包含目标区域
- 是否存在大量严重模糊帧

## 预算有限时，优先补哪三类数据

如果预算有限，不要平均投入，优先级建议如下：

### 优先级 1：新疆有真值样方

这是所有事情的基础。  
没有这部分，就无法可靠评估迁移效果。

### 优先级 2：真实设备采集的视频流

这是 future edge deployment 最值钱的数据形式。

### 优先级 3：同一样方的多视角图

它比单一正拍图更有助于：

- teacher 微调
- student 蒸馏
- 模拟视频中的视角变化

## 推荐的采集阶段

建议按三阶段推进。

### Phase 1：验证迁移可行性

- `200-400` 有真值样方
- 每个样方 `3-5` 张图或 `1` 段视频

目标：

- 判断 CSIRO 到新疆的域偏移有多严重
- 判断最优 teacher 能否直接迁移

### Phase 2：建立稳定新疆训练集

- 增加到 `600-1200` 有真值样方
- 系统覆盖区域、草地类型、季节和设备差异

目标：

- 训练新疆域 teacher
- 做可靠验证
- 为 student 蒸馏提供基础

### Phase 3：准备边缘部署数据

- 补充大量无标注视频流
- 最好来自真实设备、真实移动方式

目标：

- 训练视频流融合
- 做边缘 student 蒸馏
- 做部署前测试

## 对当前项目的直接建议

如果现在就要开始准备新疆数据，最实用的做法是：

1. 先按 `200-400` 有真值样方启动。
2. 每个样方必须至少带：
   - 多视角静态图
   - 一段短视频
   - 完整 biomass 真值
3. 尽早补真实设备视频，而不是等模型做完再补。
4. 切分时严格按 `location_id` 或 `plot_id`，不要按图片随机切。
5. 目录、命名、schema 从第一天就定死，避免后续返工。

## 总结

新疆数据采集的核心不是“再多拍一些图”，而是系统性补三种信息：

- 有真值的样方监督
- 多视角视觉变化
- 真实视频流分布

如果只补静态图，你最多是让 server-side teacher 迁移得更好一点。  
如果你同时补了视频和多视角，你才真正为：

- teacher 微调
- student 蒸馏
- `Pi 5 + Hailo-8` 视频流部署

这三件事打下基础。
